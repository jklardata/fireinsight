NFIRS → NERIS Converter
Migrate your legacy incident data to the new national standard
Why convert?
NFIRS (National Fire Incident Reporting System) is the legacy format used since 1971. NERIS became mandatory January 1, 2026. All federal grants, compliance audits, and peer benchmarks now require NERIS-formatted data.
What this tool does
  • ✓Maps NFIRS 3-digit incident codes to NERIS incident type groups
  • ✓Normalizes date/time formats to ISO 8601
  • ✓Preserves GPS coordinates, casualty counts, and property loss
  • ✓Outputs valid NERIS JSON ready to upload to the national registry
Expected CSV columns
INC_DATE, INC_NO, INC_TYPE, FDID, STATE, LATITUDE, LONGITUDE, ALARM, ARRIVAL, LU_CLEAR, FF_INJ, OTH_INJ, FF_DEATH, OTH_DEATH, PROP_LOSS, CONT_LOSS, AID_GIV, AID_REC
Standard NFIRS export from RMS systems (Firehouse, ESO, ImageTrend, etc.). Missing fields will be flagged as warnings.
After conversion: next steps
1
Download the NERIS JSON file — your converted incidents in NERIS format
2
Review warnings — records with missing coordinates, unknown codes, or malformed dates need manual review
3
Connect NERIS API — use FireInsight's NERIS API connection to submit incidents directly to the national registry
4
Run Compliance Check — verify your converted data meets all 6 NERIS module requirements
